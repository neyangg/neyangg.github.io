磨刀不误砍柴工系列(1)-懒程序员的基本款——dotfiles

Posted on 2017-01-03 | Post modified: 2018-12-21 | In 现学现卖

前言

在本系列开篇磨刀不误砍柴工系列(0)-基于CentOS/Xfce搭建最简测试环境中已经较为详细地说明了如何从零到一搭建一个可供测试的Linux开发环境，所谓磨刀不误砍柴工，在正式开始进行数据分析/挖掘的实践操作之前，我们将对整个工具链条(Linux + MySQL + Python + Hadoop)中涉及的几个重要工具的安装、配置和使用方法进行说明，以求驾轻就熟。

本篇重点说明dotfiles这一懒人神器。

什么是dotfiles

dot在英文中是「点」的意思，而对应在Linux系统中，以点开头的文件通常是隐藏文件，保存着软件的配置信息，系统就是通过读取一个个这样的文件实现软件的设置和自定义功能的。

认识到这点，那接下来就很容易理解了，当拿到一台新的机器，或者因为系统奔溃等原因不得不重新安装操作系统（这在Windows下很常见），意味着我们要重新进行常用软件的安装和配置，通常这是一项巨大的工程。有人就想到（太聪明了），是不是可以把这些配置文件集中起来进行统一的管理和备份呢？这就是dotfiles的解决问题的思路，

  1. 在HOME目录下创建一个dotfiles文件夹，里面保存了所有常用软件的配置文件，例如.vimrc，.zshrc等
  2. 在常用软件目录下创建指向dotfiles内配置文件的软链接
  3. 对dotfiles进行统一的管理和备份

当然，如果手动去进行上述操作还是很麻烦，更进阶的方法是把以上所有的操作都写进Shell脚本内，重装系统后，运行脚本，一键配置，So easy！

dotfiles配置

接下来以我的个人配置为例进行说明，实际上，dotfiles的管理功能十分强大，可以对系统层面进行同样的配置管理，但一方面我并不需要这样的功能，另外高度的定制化也就意味着可迁移性变差，更换一个发行版可能就不再适用，违背了dotfiles的初衷，所以这里我只用其来进行Zsh、Tmux、Vim的安装、配置管理，以及一些环境变量和别名的定义。

与前面提到的思路相同，我在HOME目录下建立一个文件夹，命名为.dotfiles（会默认隐藏，防止误操作），然后将配置好的软件配置文件拷贝到此目录下，命名为tmux.conf, vimrc, zshrc。最后使用Shell脚本进行对应软件的安装和软链。

说明，我的脚本是基于Brain Cain大神的starter-dotfiles进行修改得到的，有兴趣可以自行研究，定义属于自己的脚本。

dotfiles使用

接下来就简单了，将整个.dotfiles文件夹托管在github上，重装系统后，git clone - ./install.sh - enjoy! 简直太方便了有木有！

现在你可以把更多的时间浪费在更有意义的事情上了，果然懒惰是人类进步的阶梯~~
